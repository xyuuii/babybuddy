# Security Policy

## Supported Versions

The project currently starts at version `1.0`.

## Reporting a Vulnerability

Please report security issues through GitHub Security Advisories when available, or by contacting the repository owner privately if the issue contains sensitive details. Use public GitHub issues only for non-sensitive security hardening discussions.

Do not post real API keys, NAS credentials, private family media, or private server URLs in public issues.

## Security Notes

- Prefer HTTPS WebDAV endpoints.
- Use a dedicated NAS/WebDAV user with limited folder permissions.
- Keep AI provider API keys local.
- Do not commit release signing keys.
- Review dependencies before publishing production builds.
